## 🎉 Features

- Support for multiple built-in data structures
- High throughput, low latency, and efficient batch data writing
- Support for disk data storage and garbage data recycling
- Support for static encryption and compression of disk data
- IP whitelist to ensure secure access to data
- Operate data through RESTful API

## 🚀 Quick Start

Use Docker to quickly deploy the [`wiredb:latest`](https://hub.docker.com/r/auula/wiredb) image and test the service provided by WireDB:

```bash
docker pull auula/wiredb:latest
```

Run the WireDB container and map the port to your host network:

```bash
docker run -p 2668:2668 auula/wiredb:latest
```

WireDB uses RESTful API for data interaction, so any HTTP-compatible client can access and operate the WireDB instance. To authorize, include an `Auth-Token` in the request header. This token is auto-generated by the WireDB process and can be found in the container logs:


## 🕹️ RESTful API

WireDB provides data interaction interfaces via RESTful API over HTTP. It's recommended to use `curl` for testing. WireDB supports several abstract data structures such as `Table`, `List`, `ZSet`, `Set`, `Number`, and `Text`.

The `Table` structure is similar to JSON or any semi-structured data with key-value mappings, like structs or classes in programming. Here's an example table with a `ttl` field (time to live, deleted after 120 seconds):

```json
{
    "table": {
        "is_valid": false,
        "items": [
            {
                "id": 1,
                "name": "Item 1"
            },
            {
                "id": 2,
                "name": "Item 2"
            }
        ],
        "meta": {
            "version": "2.0",
            "author": "Leon Ding"
        }
    },
    "ttl": 120
}
```

Store this data using the PUT method like so:

```bash
curl -X PUT http://localhost:2668/table/key-01 -v \
     -H "Content-Type: application/json" \
     -H "Auth-Token: T9EHAvi5dcIpPK9G#ADlVj4NB" \
     --data @tests/table.json
```

To fetch data:

```bash
curl -X GET http://localhost:2668/table/key-01 -v \
     -H "Auth-Token: T9EHAvi5dcIpPK9G#ADlVj4NB"
```

To delete data:

```bash
curl -X DELETE http://localhost:2668/table/key-01 -v \
     -H "Auth-Token: T9EHAvi5dcIpPK9G#ADlVj4NB"
```

More complex query/update operations will be supported in future versions. See the [official docs](https://docs.wiredb.org) for examples of other data types.
